Markov Decision Process - given a current state and action to take, translates this into a new state.

Markovian property - history doesn't matter, only the current state matters

  • History can be wrapped up in the current state, or the current state can be something like (a1, a2, a3, etc..)

MDPs do not have to end.

States: S

Model: T(s, a, s') ~ Pr(s' | s,a) (T = Transition model, a "matrix" of states and actions with probabilities that you will end up in a particular state given your current state and action taken.

Actions: A(s), A

Reward: R(s), R(s,a), R(s,a,s')

  • Rewards can just be taking a state, taking a specific action when in a state, or being in as state, taking a specific action, resulting in a specific state
  • These are all mathematically equivalent, and all these can collapse into R(s)

Policy: π\piπ(s) → a the function that tells you what action you should take based on your state.

  • Solution to a MDP
  • Optimal policy π∗\pi^*π∗ is the policy that results in the highest reward from that state
    • Always choose the highest reward for the optimal policy. If the rewards
  • Policy calculates the optimal policy by analysing the <s,a,r> sequences

MDP are "stationary", meaning that they don't change over time. The probabilities from a given state, action pairs will not change based on whatever previous actions taken.

Finding Policies

There are n equations for n states in n unknowns in Bellman's equations for all states

How do we find the policies from the utility functions?

  • start with arbitrary utility functions (immediate rewards only)?
  • update utilities based on neighbors
  • repeat until convergence

Value Iteration

Equation which models this recursive process, recursive Bellman Equation.

Ut+1(s)=R(s)+γmax⁡a∑s′T(s,a,s′)Ut(s′)U_{t+1}(s) = R(s) + \gamma\max_a\sum\limits_{s'} T(s,a,s') U_t(s')Ut+1​(s)=R(s)+γmaxa​s′∑​T(s,a,s′)Ut​(s′)

This is pretty much a loop where each state figures out it its reward recursively, and then adds in the immediate reward

Note how the U1(s) = 0.36 by taking R(s) = -.04 and then gamma = 1/2 of the sum of the probabilities * the currently known utility scores for the square below and the current square (because going up just bumps into the wall), which means it equals 0.36.

BUT, the next iteration of U2(s) where s = (0,2) will increase because the utilities will now include some values for the down square and the starting square... Hence the iterative nature of value iteration.

Algorithm:

  • start with some policy π0\pi_0π0​ where it's just random actions
  • evaluate the UtU_tUt​ for this policy (Ut=Uπt(U_t = U^{\pi_t}(Ut​=Uπt​)

The equation now becomes

Ut(s)=R(s)+γ∑s′T(s,πt(s),s′)Ut(s′)U_t(s) = R(s) + \gamma \sum \limits_{s'} T(s,\pi_t(s), s') U_t(s')Ut​(s)=R(s)+γs′∑​T(s,πt​(s),s′)Ut​(s′)

Notice there is no max since we're not following all actions, we're only following the action provided by the policy in the T (transition) function.

Because the max is missing, it becomes linear, but it's still complicated. And it's guaranteed to converge.

Policy Iteration vs Value Iteration

Value iteration makes more iterations but they require less expensive computations,

Policy iteration makes fewer iterations, but each one is more expensive

Value iterations makes "jumps" in value space

Policy iterations makes "jumps" in policy space

Summary

MDP - States, Rewards, Actions, Transitions, Discount (gamma)

States, Actions, Transitions represent the physical world

Rewards and discount represent the task description we're trying to accomplish

Utilities are long term rewards - a group of rewards, whereas rewards are just the immediate for the current state

Stationary - things don't change

Bellman equation underlies all of it

MDP is a model based learning, not true reinforcement learning

RL doesn't necessarily need to know everything like the MDP does.
